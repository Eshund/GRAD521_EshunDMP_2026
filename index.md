# Data Description
The data for my project consist of experimental measurements collected from cement based material specimens under controlled laboratory conditions. Multiple datasets will be generated during the project. One dataset includes numerical measurements collected over time from laboratory instruments used to evaluate transport and durability related properties. These data will be stored in spreadsheet formats such as CSV and Excel files. A second dataset includes image based data collected during moisture movement and absorption testing. These files will include raw images and processed outputs used for analysis. Another dataset includes measurements from environmental exposure testing, such as changes in material performance under controlled conditions. These data may include numerical results, calculated values, and photographs. Additional datasets include measurements related to pore structure, moisture behavior, and thermal analysis. These will be stored as numerical files, graphs, and processed results. Metadata will also be created for all datasets. This includes sample identification codes, test dates, instrument settings, operator notes, and file descriptions needed to understand and organize the data. Most numerical data will be stored as CSV or Excel files. Images will be stored in common formats such as PNG or TIFF. The total expected size of the complete dataset is approximately 5 to 10 GB, depending on the number of tests and image files generated during the project.

# Roles and responsibilities
The main responsibility for implementing the Data Management Plan belongs to the Principal Investigator (PI), who oversees the research project and ensures that data management practices meet university, department, and research group expectations. The graduate student has the main day to day responsibility for carrying out the plan. This includes organizing files, following naming conventions, maintaining folders, recording laboratory notes, and ensuring that data are uploaded and stored correctly. The graduate student is also responsible for creating metadata, which means documenting how the data were collected, processed, analyzed, and interpreted so that others can understand and reuse the information later. Data collection and generation are mainly handled by the graduate student through laboratory experiments. This includes preparing specimens, conducting tests, recording measurements, and saving outputs from instruments. Instrument maintenance is shared between the graduate student, laboratory staff, and technical personnel who help ensure that equipment such as resistivity meters, environmental chambers, and imaging systems are functioning properly. Quality control is handled through regular checks of data accuracy, calibration of instruments, review of unusual results, and comparison of repeated tests. Data analysis is mainly performed by the graduate student, while the PI provides technical guidance and reviews interpretations. Finalized datasets and completed project files are prepared for long term archiving once the work is complete. If a person with data management responsibilities leaves the project, the work can continue because files are stored in shared systems and documentation is maintained. Experimental procedures, folder structures, file names, and notes are kept organized so another researcher can understand the status of the project. This reduces the risk of losing knowledge when personnel changes occur.

# Data standards and metadata
To keep project data organized and consistent, standard file naming and folder structures will be used throughout the research project. File names will include key information such as test type, sample identification code, date, and version number when needed. An example format is: TestType_S01_2026_05_15_v1.csv or TestType_S03_28days_2026_06_10.xlsx. This system will make it easier to identify files, locate the most recent version, and reduce confusion during analysis. Numerical data will be stored in commonly used formats such as CSV and Excel files. Image data will be stored as PNG, or JPEG files. Reports and summaries will be saved as PDF and Word documents. Metadata will be created for each dataset to help users understand and interpret the files. This will include sample identification codes, test dates, operator initials, file descriptions, instruments used, general testing conditions and notes related to data processing. Processed files will also include information about the software or methods used for analysis. A README file will be included in major folders to explain file contents, abbreviations, units, and folder organization. Additional notes will be recorded in laboratory notebooks or digital logs during the project. Version control will be applied to scripts, reports, and processed data files. Raw data files will be preserved in their original form and will not be overwritten. If changes are made during processing, updated versions will be saved with revised file names. These practices will improve organization, traceability, and long term usability of the project data.

# Storage and security
During the project, data will be stored primarily in the research group’s Box cloud storage system. Box is the main shared location for active project files such as spreadsheets, images, reports, scripts, and processed results. Access will be limited to authorized research group members and approved collaborators. Box also includes file recovery and version history features, which help restore deleted or overwritten files. Temporary working files may be stored on laboratory computers or personal devices during active testing or analysis. Important files will be transferred to Box regularly to reduce the risk of data loss. Devices used for research will be protected with passwords and university approved security practices. The project does not involve human subjects or confidential personal information. However, all research data will still be handled according to university standards for responsible data management and security.

# Access and data sharing
During the active research phase, access to project data will be limited to authorized members of the research group. Internal working files, draft analyses, and unpublished results will remain restricted until approved for release by the Principal Investigator. After publication or project completion, selected datasets may be shared with collaborators or deposited in an appropriate repository, depending on advisor approval, sponsor requirements, and university policies. Shared datasets will include supporting documentation such as metadata, README files, and file descriptions so that other users can understand and reuse the data properly. Any data shared publicly will exclude unnecessary internal notes or materials not intended for external use.

# Archiving and preservation
A multi copy backup strategy will be used throughout the project. One copy of active data may remain on the laboratory or instrument computer during collection, while the main working copy will be stored on Box. Additional local backup copies will be kept on secure storage devices when needed. Box version history and recovery tools will help restore previous versions of files if accidental deletion or unwanted changes occur. Regular uploads will be performed after experiments and major analysis updates. At the end of the project, final datasets, processed results, scripts, reports, and supporting documentation will remain in the shared Box folders so the research group retains access. Files will be organized clearly with consistent naming, metadata, and documentation to support long term preservation and future reuse.
